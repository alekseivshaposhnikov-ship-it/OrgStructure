| Элемент | Размер | Насыщенность |
| --- | ---: | ---: |
| Заголовок приложения | `18px` | `600–700` |
| Заголовок модального окна | `20–24px` | `700` |

## 3. Компоненты

### Кнопки

Основная кнопка:

* фон `#4f46e5`;
* текст `#ffffff`;
* скругление `8px`;
* минимальная высота `36px`;
* hover-фон `#4338ca`.

Вторичная кнопка:

* светлый или белый фон;
* текст основного акцента;
* граница `#d0d5dd`;
* используется для сброса, отмены и вторичных действий.

Иконка «Что нового»:

* размер кликабельной области `32 × 32px`;
* круглая форма;
* белый фон;
* акцентная иконка;
* индикатор NEW размещается в правом верхнем углу.

### Карточки подразделений

Карточка содержит название подразделения, руководителя, должность и численность.

Правила:

* скругление `8–14px`;
* граница акцентного или нейтрального цвета;
* название — основной визуальный акцент;
* численность отделена визуально и не должна конкурировать с названием.

### Карточки сотрудников

Карточка содержит ФИО, должность и проект при наличии.

Правила:

* белый фон;
* нейтральная граница;
* карточка кликабельна и открывает детальную информацию;
* контекстное меню не должно мешать клику по карточке.

### Карточки вакансий

Карточка содержит метку `Вакансия`, должность и проект при наличии.

Правила:

* светло-синий фон;
* синяя граница;
* детальная карточка сотрудника не открывается.

### Модальные окна

Правила:

* затемненный backdrop;
* белая карточка;
* скругление не менее `12px`;
* кнопка закрытия в правом верхнем углу;
* широкий вариант используется для сравнения и журнала обновлений.

### Панели

Левая панель:

* фиксированная ширина около `340px`;
* белый фон;
* вертикальный скролл;
* содержит заголовок, управление и дерево.

Правая панель изменений:

* используется для списка операций сценария;
* может скрываться на средних экранах.

### Таблицы

Таблицы используются для сравнения AS IS / TO BE.

Правила:

* четкие заголовки столбцов;
* нейтральные границы;
* числовые изменения показываются в отдельном столбце.

### Индикаторы

Индикатор NEW:

* красный фон `#f04438`;
* белый текст;
* компактная форма pill;
* не должен занимать отдельную строку интерфейса.

## 4. Правила отступов и размеров

| Правило | Значение |
| --- | ---: |
| Базовый внутренний отступ панели | `16px` |
| Отступ между контролами | `12px` |
| Скругление базовых элементов | `8px` |
| Скругление крупных карточек | `12–14px` |
| Минимальная кликабельная область | `32 × 32px` |
| Предпочтительная высота input/select | `36px` |
| Тень карточек | `0 1px 3px rgba(0,0,0,0.05)` |
| Тень акцентных карточек | `0 2px 6px rgba(0,0,0,0.1)` |

## 5. Правила интерфейса

* Главные действия должны быть визуально заметнее вторичных.
* Сервисные действия, например журнал обновлений, размещаются компактно и не конкурируют с основными сценариями пользователя.
* Сценарное моделирование по умолчанию свернуто, чтобы не занимать рабочую область.
* Экспорт PDF должен сохранять направление дерева и порядок уровней экранной диаграммы.
* Левое дерево отображает только подразделения и не перегружается сотрудниками, вакансиями и численностью.
* Документация изменений должна быть краткой: версия, дата, добавлено, исправлено, реализовано.
