# CR-002. Улучшение пользовательского интерфейса и документации

Статус: Новый

Приоритет: Средний

Дата создания: 23.06.2026

---

## Цель

Повысить удобство использования приложения, улучшить навигацию, привести документацию к единому стилю и устранить расхождения между экранным отображением и экспортом PDF.

---

## Изменение 1. Переработка элемента «Что нового»

### Текущая проблема

Раздел «Что нового» реализован в виде отдельной кнопки.

Кнопка визуально занимает много места в панели управления и конкурирует за внимание с основными элементами интерфейса.

Черное оформление кнопки выглядит тяжеловесно и не соответствует общей визуальной стилистике приложения.

### Требование

Не использовать отдельную крупную кнопку.

Разместить доступ к журналу обновлений на уровне заголовка:

```text
Организационная структура
```

в виде компактной иконки.

Примеры:

```text
ⓘ
🔔
🛈
```

или собственной SVG-иконки.

### Ожидаемое поведение

Пользователь видит компактную иконку рядом с названием приложения.

При нажатии открывается журнал обновлений.

При наличии непросмотренных обновлений отображается индикатор новой версии.

## Изменение 2. Сворачивание блока сценарного моделирования

### Текущая проблема

Блок сценарного моделирования занимает значительную область интерфейса.

На текущем этапе функциональность используется ограниченно и содержит небольшое количество полезной информации.

### Требование

По умолчанию блок должен быть свернут.

Отображается только заголовок.

Пример:

```text
▶ Сценарий моделирования
```

После раскрытия:

```text
▼ Сценарий моделирования
```

отображается полный набор элементов управления.

### Дополнительно

Состояние блока сохраняется в:

```text
localStorage
```

Пример ключа:

```text
orgScenarioPanelCollapsed
```

## Изменение 3. Переименование дизайна карточек

### Текущая проблема

Название:

```text
Вариант 3 — руководитель
```

не объясняет визуальную особенность шаблона.

### Требование

Заменить название на более понятное.

Предлагаемые варианты:

```text
Карточка с руководителем
```

или

```text
Руководитель с аватаром
```

или

```text
Профиль руководителя
```

Предпочтительный вариант:

```text
Профиль руководителя
```

## Изменение 4. Корректный экспорт структуры в PDF

### Текущая проблема

Экспортируемая структура визуально отличается от структуры, отображаемой пользователю на экране.

В PDF элементы раскладываются в широкую горизонтальную схему.

Пользователь ожидает получить структуру максимально близкую к отображаемой диаграмме.

### Требование

Экспорт должен сохранять логику расположения элементов экранной диаграммы.

Допускаются:

* автоматическое масштабирование;
* перенос на несколько страниц;
* уменьшение размеров карточек.

Не допускается:

* изменение направления дерева;
* изменение порядка уровней;
* перестроение структуры в другой визуальный формат.

### Результат

PDF должен быть визуально максимально близок к отображаемой пользователю диаграмме.

## Изменение 5. Создание брендбука интерфейса

### Цель

Зафиксировать единые правила визуального оформления приложения.

### Требование

Создать отдельный документ:

```text
ui-brandbook.md
```

Документ должен содержать:

### Цвета

* основные цвета интерфейса;
* акцентные цвета;
* цвета сценарных изменений;
* цвета состояний;
* цвета текста;
* цвета фона.

### Типографика

* основной шрифт;
* размеры заголовков;
* размеры текста;
* размеры подписей;
* размеры элементов интерфейса.

### Компоненты

* кнопки;
* карточки подразделений;
* карточки сотрудников;
* карточки вакансий;
* модальные окна;
* панели;
* таблицы;
* индикаторы.

### Правила

* отступы;
* скругления;
* тени;
* размеры иконок;
* размеры кликабельных областей.

## Изменение 6. Упрощение раздела «Что нового»

### Текущая проблема

Описание функционала содержит большое количество повторяющихся формулировок и общих пояснений.

Из-за этого становится сложно быстро понять:

* что изменилось;
* когда изменилось;
* зачем изменилось.

### Требование

Переработать описание раздела.

Оставить только:

* версию;
* дату;
* список изменений;
* список исправлений;
* реализованные пожелания.

Пример:

Версия 1.4.0

Дата: 24.06.2026

Добавлено:

* Журнал обновлений.

Исправлено:

* Экспорт PDF.

Реализовано:

* Отображение проектов сотрудников.

### Результат

Раздел должен использоваться как практический журнал изменений, а не как подробная документация по функционалу.
