# Функциональное описание приложения «Org Structure Viewer»

## 1. Назначение

Приложение предназначено для визуализации организационной структуры холдинга LEGENDA.

Пользователь может:

* просматривать дерево подразделений;
* выбирать подразделение в левом дереве;
* видеть выбранное подразделение и его подчиненные элементы в организационной диаграмме;
* отображать подразделения, сотрудников и вакансии;
* скрывать и показывать вакансии;
* анализировать численность сотрудников и вакансий;
* открывать детальную карточку сотрудника;
* переключать дизайн карточек подразделений;
* экспортировать текущую диаграмму в PDF.

## 2. Источник данных

Данные загружаются из API:

```http
GET /api/getDepartmentVacancy
```

В режиме локальной разработки запрос `/api` проксируется через Vite на корпоративный API.

Если загрузка не удалась, пользователь видит сообщение об ошибке.

## 3. Корневой узел

Приложение создает виртуальный корневой узел:

```text
Холдинг LEGENDA
```

Он не приходит из API, а формируется на стороне приложения.

Для корневого узла задаются:

* название холдинга;
* руководитель: Селиванов Василий Геннадьевич;
* должность: Генеральный директор;
* суммарная численность сотрудников;
* суммарное количество вакансий;
* общая численность с учетом вакансий.

## 4. Левая панель навигации

В левой панели отображается дерево подразделений.

В дереве отображаются только:

* виртуальный корневой узел;
* подразделения;
* вложенные подразделения.

В левом дереве не отображаются:

* сотрудники;
* вакансии;
* руководители;
* численность.

Пользователь может:

* раскрывать и сворачивать подразделения по стрелке;
* раскрывать и сворачивать подразделения двойным кликом по названию;
* выбирать подразделение кликом по названию.

Выбранное подразделение подсвечивается.

## 5. Организационная диаграмма

В правой части отображается организационная диаграмма выбранного узла.

Диаграмма включает:

* выбранное подразделение;
* дочерние подразделения;
* сотрудников подразделений;
* вакансии, если включен режим показа вакансий.

После выбора подразделения диаграмма перестраивается и показывает выбранный узел со всеми подчиненными элементами.

## 6. Карточка подразделения

Карточка подразделения содержит:

* название подразделения;
* ФИО руководителя;
* должность руководителя;
* численность.

Если название подразделения отсутствует, используется:

```text
Без названия
```

Если руководитель отсутствует, используется:

```text
Нет руководителя
```

Должность руководителя сокращается до части перед символом `/`.

## 7. Сотрудники

Сотрудники отображаются только в организационной диаграмме.

Карточка сотрудника содержит:

* ФИО;
* должность.

Если ФИО отсутствует, используется:

```text
Сотрудник
```

Сотрудники попадают в отображение и расчет численности только если поле `count` после преобразования в число больше или равно `1`.

Руководитель подразделения исключается из списка сотрудников подразделения, чтобы не дублироваться в диаграмме.

Сотрудник считается руководителем, если:

* совпадает `id` сотрудника и руководителя;
* либо совпадает ФИО сотрудника и руководителя.

## 8. Вакансии

Вакансии отображаются только в организационной диаграмме и только при включенном переключателе:

```text
Показывать вакансии
```

Карточка вакансии содержит:

* текст `Вакансия`;
* должность вакансии.

Должность вакансии также сокращается до части перед символом `/`.

Вакансии не открывают детальную карточку.

## 9. Переключатель вакансий

По умолчанию вакансии включены.

При включенном режиме:

* вакансии отображаются в диаграмме;
* численность подразделений отображается с учетом вакансий;
* используется показатель `totalWithVacancies`;
* в текущем дизайне численность подсвечивается синим.

При выключенном режиме:

* вакансии скрываются;
* отображается только фактическая численность сотрудников;
* используется показатель `staffCount`.

## 10. Расчет численности

Для каждого подразделения рассчитываются:

```text
staffCount
vacancyCount
totalWithVacancies
```

`staffCount` включает сотрудников текущего подразделения и всех дочерних подразделений.

`vacancyCount` включает вакансии текущего подразделения и всех дочерних подразделений.

`totalWithVacancies` рассчитывается так:

```text
totalWithVacancies = staffCount + vacancyCount
```

В левой панели численность не отображается.

## 11. Сортировка сотрудников и вакансий

Сотрудники и вакансии сортируются перед отображением.

Сначала отображаются сотрудники, затем вакансии.

Основной порядок сотрудников определяется полем `sub_level`: чем меньше значение, тем выше сотрудник.

Если `sub_level` отсутствует, используется резервная сортировка по ключевым словам в должности:

| Ключевое слово | Приоритет |
| -------------- | --------: |
| директор       |         1 |
| руководитель   |         2 |
| начальник      |         3 |
| лидер / lead   |         4 |
| ведущий        |         5 |
| старший        |         6 |
| главный        |         6 |

Если приоритет совпадает, используется сортировка по ФИО или должности в русской локали.

## 12. Детальная карточка сотрудника

Детальная карточка открывается по клику на сотрудника в организационной диаграмме.

Карточка не открывается для вакансий.

В карточке могут отображаться:

* фотография;
* ФИО;
* должность;
* проект;
* телефон;
* email;
* тип занятости;
* статус.

Если фотография отсутствует, отображается заглушка.

Email отображается как ссылка `mailto:`.

Карточку можно закрыть:

* кнопкой `×`;
* кликом по затемненной области вне карточки.

## 13. Поддерживаемые поля сотрудника

Проект определяется из одного из полей:

```text
project
project_name
projectName
project_title
```

Телефон определяется из одного из полей:

```text
phone
phone_number
mobile
work_phone
```

Фотография определяется из одного из полей:

```text
photo
photo_url
avatar
avatar_url
```

Тип занятости берется из поля:

```text
type_employment
```

Статус берется из поля:

```text
state
```

## 14. Сокращение должностей

Для руководителей, сотрудников и вакансий используется сокращенное отображение должности.

Если в должности есть символ `/`, отображается только часть до него.

Пример:

```text
Руководитель отдела / Юридический департамент
```

отображается как:

```text
Руководитель отдела
```

## 15. Дизайны карточек подразделений

В приложении доступны три варианта дизайна карточек подразделений:

```text
Текущий
Вариант 2 — цветная шапка
Вариант 3 — руководитель
```

Выбранный дизайн сохраняется в `localStorage` по ключу:

```text
orgCardDesign
```

При повторном открытии приложения используется сохраненный вариант.

### Текущий дизайн

Содержит:

* название подразделения;
* руководителя;
* должность руководителя;
* численность.

При включенных вакансиях численность подсвечивается синим.

### Вариант 2 — цветная шапка

Содержит:

* цветную шапку с названием подразделения;
* ФИО руководителя;
* должность руководителя;
* численность в нижнем блоке.

Численность отображается в формате:

```text
N сотрудников
```

### Вариант 3 — руководитель

Содержит:

* аватар с инициалами руководителя;
* название подразделения;
* ФИО руководителя;
* должность руководителя;
* численность.

Если ФИО руководителя отсутствует, в аватаре отображается:

```text
—
```

## 16. Экспорт в PDF

Пользователь может экспортировать текущую организационную диаграмму в PDF по кнопке:

```text
📄 Экспорт в PDF
```

Экспортируется именно текущая отображаемая диаграмма.

Во время экспорта:

* кнопка временно блокируется;
* текст кнопки меняется на `Экспорт...`;
* поверх SVG-узлов временно добавляются рамки для корректного отображения карточек;
* диаграмма преобразуется в изображение;
* создается PDF;
* файл сохраняется под именем:

```text
orgchart.pdf
```

Ориентация PDF выбирается автоматически:

* `landscape`, если ширина изображения больше или равна высоте;
* `portrait`, если высота больше ширины.

Если диаграмма отсутствует, пользователь видит:

```text
Нет диаграммы для экспорта
```

Если экспорт завершился ошибкой, пользователь видит сообщение об ошибке.

## 17. Состояния загрузки и ошибок

При запуске отображается сообщение:

```text
Загрузка организационной структуры...
```

Если API вернул пустой результат:

```text
Не удалось загрузить структуру.
```

Если данные не удалось загрузить:

```text
Не удалось загрузить организационную структуру. Проверьте доступность сервера.
```

Если произошла ошибка инициализации:

```text
Произошла ошибка. Обновите страницу.
```

## 18. Адаптивность

При ширине экрана до 700 пикселей:

* интерфейс перестраивается из горизонтального в вертикальный;
* боковая панель располагается сверху;
* диаграмма располагается снизу;
* высота боковой панели ограничивается;
* область диаграммы занимает оставшуюся часть экрана.

## 19. Технические особенности

Приложение использует:

* Vite;
* D3;
* d3-flextree;
* d3-org-chart;
* html-to-image;
* jsPDF.

Основной сценарий работы реализован через:

* загрузку данных из API;
* преобразование API-ответа во внутреннюю древовидную структуру;
* построение левого дерева подразделений;
* преобразование выбранного узла в плоский список для `d3-org-chart`;
* отрисовку диаграммы;
* обработку кликов по сотрудникам;
* экспорт текущей диаграммы в PDF.
