# Функциональное описание приложения «Org Structure Viewer»

## 1. Общее назначение

### 1.1 Назначение системы

Приложение предназначено для визуализации организационной структуры холдинга LEGENDA в виде интерактивного дерева подразделений и организационной диаграммы.

Система позволяет:

* просматривать структуру компании на любом уровне вложенности;
* отображать подразделения, сотрудников и вакансии в организационной диаграмме;
* использовать левое дерево подразделений для навигации по структуре;
* анализировать численность подразделений;
* получать детальную информацию о сотрудниках;
* скрывать и показывать вакансии;
* переключать дизайн карточек подразделений;
* экспортировать текущую диаграмму в PDF.

### 1.2 Источник данных

Данные загружаются в режиме реального времени из корпоративного API:

```http
GET /api/getDepartmentVacancy
```

В режиме локальной разработки запрос проксируется через Vite на корпоративный API.

---

# 2. Отображение организационной структуры

## 2.1 Корневой узел

Структура отображается начиная с виртуального корневого узла:

**Холдинг LEGENDA**

В организационной диаграмме для корневого узла отображаются:

* наименование холдинга;
* ФИО руководителя;
* должность руководителя;
* численность сотрудников или общая численность с учетом вакансий в зависимости от режима отображения вакансий.

По умолчанию руководителем корневого узла является:

**Селиванов Василий Геннадьевич**
*Генеральный директор*

Корневой узел формируется на стороне приложения и не является отдельным узлом, полученным из API.

---

## 2.2 Левая панель с деревом подразделений

Левая панель предназначена только для навигации по иерархии подразделений.

В дереве отображаются только:

* корневой узел;
* подразделения;
* вложенные подразделения.

В левой панели не отображаются:

* руководители подразделений;
* сотрудники;
* вакансии;
* численность подразделений.

Дерево используется исключительно для выбора подразделения и навигации по структуре.

---

## 2.3 Карточка подразделения в организационной диаграмме

Каждое подразделение в правой части приложения отображается отдельной карточкой.

Карточка содержит:

* название подразделения;
* ФИО руководителя;
* должность руководителя;
* численность подразделения.

Если руководитель не указан, в карточке отображается:

```text
Нет руководителя
```

Если название подразделения отсутствует, используется значение:

```text
Без названия
```

### Пример отображения

```text
Юридический департамент

👤 Максимова Мария Сергеевна
Руководитель отдела

15
```

---

## 2.4 Отображение руководителя

Руководитель подразделения отображается только в карточке подразделения в организационной диаграмме.

При загрузке данных выполняется автоматическое исключение руководителя из общего списка сотрудников подразделения.

Сотрудник считается руководителем, если:

* совпадает идентификатор сотрудника и руководителя;
* либо совпадает ФИО сотрудника и руководителя.

Это исключает дублирование руководителя в списках сотрудников организационной диаграммы.

---

## 2.5 Сокращение должностей

Для отображения должностей используется сокращенное значение должности.

Если в значении должности присутствует символ `/`, отображается только часть строки до этого символа.

### Пример

```text
Руководитель отдела / Юридический департамент
```

отображается как:

```text
Руководитель отдела
```

Это правило применяется к:

* должности руководителя подразделения;
* должности сотрудника;
* должности вакансии.

---

# 3. Управление отображением вакансий

## 3.1 Переключатель вакансий

В боковой панели доступен переключатель:

```text
☑ Показывать вакансии
```

По умолчанию переключатель включен.

---

## 3.2 Режим «Показывать вакансии»

При включенном переключателе:

* вакансии отображаются в организационной диаграмме;
* численность подразделений рассчитывается с учетом вакансий;
* численность в карточках подразделений в правой части подсвечивается синим цветом в текущем дизайне карточек.

Используется показатель:

```text
totalWithVacancies
```

В левой панели дерево не меняет состав отображаемых элементов: в нем по-прежнему отображаются только подразделения.

---

## 3.3 Режим «Скрывать вакансии»

При отключенном переключателе:

* вакансии скрываются из организационной диаграммы;
* отображается только фактическая численность сотрудников;
* численность в карточках подразделений отображается обычным цветом.

Используется показатель:

```text
staffCount
```

В левой панели дерево не меняет состав отображаемых элементов: в нем по-прежнему отображаются только подразделения.

---

# 4. Расчет численности

Для каждого подразделения рассчитываются следующие показатели.

## 4.1 Фактическая численность

Количество сотрудников, прошедших фильтрацию по полю `count`:

```text
staffCount
```

Включает:

* сотрудников текущего подразделения;
* сотрудников всех дочерних подразделений.

В расчет попадают только сотрудники, у которых значение поля `count` после преобразования в число больше либо равно `1`.

```text
count >= 1
```

Сотрудники с меньшим значением `count` не включаются в список сотрудников и не участвуют в расчете фактической численности.

---

## 4.2 Количество вакансий

Количество открытых вакансий:

```text
vacancyCount
```

Включает:

* вакансии текущего подразделения;
* вакансии дочерних подразделений.

---

## 4.3 Общая численность

Количество сотрудников и вакансий:

```text
totalWithVacancies = staffCount + vacancyCount
```

---

## 4.4 Отображение численности

Численность отображается только в карточках подразделений организационной диаграммы.

В левой панели численность не отображается.

Если включен режим «Показывать вакансии», численность в правой части отображается по показателю `totalWithVacancies`.

Если режим «Показывать вакансии» отключен, численность отображается по показателю `staffCount`.

В текущем дизайне карточек при включенном режиме «Показывать вакансии» численность подсвечивается синим цветом.

---

# 5. Сортировка сотрудников

## 5.1 Сортировка по уровню должности

Сотрудники отображаются не в порядке получения из API, а в соответствии с уровнем должности.

Основным источником порядка является поле:

```text
sub_level
```

Чем меньше значение, тем выше сотрудник располагается в списке.

Пример порядка отображения:

1. Директор
2. Руководитель
3. Начальник
4. Лидер направления
5. Ведущий специалист
6. Старший специалист
7. Специалист

---

## 5.2 Резервная сортировка

Если уровень должности отсутствует, используется анализ наименования должности.

Приоритеты:

| Ключевое слово | Приоритет |
| -------------- | --------- |
| директор       | 1         |
| руководитель   | 2         |
| начальник      | 3         |
| лидер / lead   | 4         |
| ведущий        | 5         |
| старший        | 6         |
| главный        | 6         |

Если ключевые слова не найдены, используется резервный приоритет:

```text
100
```

Если приоритеты совпадают, элементы дополнительно сортируются по ФИО или должности в русской локали.

---

## 5.3 Сортировка вакансий

Вакансии всегда отображаются после сотрудников подразделения независимо от уровня должности.

---

# 6. Отображение сотрудников и вакансий

## 6.1 Отображение сотрудников

Сотрудники отображаются только в организационной диаграмме.

В левой панели сотрудники не отображаются.

Карточка сотрудника содержит:

* ФИО;
* должность.

Если ФИО сотрудника отсутствует, используется значение:

```text
Сотрудник
```

### Пример

```text
Иванов Иван Иванович
Ведущий специалист
```

---

## 6.2 Отображение вакансий

Вакансии отображаются только в организационной диаграмме при включенном режиме «Показывать вакансии».

В левой панели вакансии не отображаются.

Карточка вакансии содержит:

* признак «Вакансия»;
* название должности.

### Пример

```text
Вакансия

Ведущий юрисконсульт
```

---

# 7. Детальная карточка сотрудника

## 7.1 Открытие карточки

Карточка открывается при нажатии на сотрудника в организационной диаграмме.

Так как сотрудники не отображаются в левом дереве, из левой панели карточка сотрудника не открывается.

Карточка не открывается для вакансий.

---

## 7.2 Отображаемые данные

Карточка сотрудника может содержать:

* фотографию;
* ФИО;
* должность;
* проект;
* телефон;
* email;
* тип занятости;
* статус сотрудника.

Если фотография отсутствует, отображается графическая заглушка с иконкой сотрудника.

Email отображается как ссылка для отправки письма.

---

## 7.3 Проект сотрудника

Информация о проекте загружается из API.

Поддерживаются поля:

* `project`
* `project_name`
* `projectName`
* `project_title`

Проект отображается исключительно в карточке сотрудника и не участвует в формировании структуры подразделений.

---

## 7.4 Контактные данные

Поддерживается загрузка телефона из следующих полей:

* `phone`
* `phone_number`
* `mobile`
* `work_phone`

Поддерживается загрузка фотографии из следующих полей:

* `photo`
* `photo_url`
* `avatar`
* `avatar_url`

---

## 7.5 Тип занятости и статус

Тип занятости сотрудника загружается из поля:

```text
type_employment
```

Статус сотрудника загружается из поля:

```text
state
```

В карточке сотрудника эти данные отображаются как:

```text
Тип занятости
Статус
```

---

## 7.6 Закрытие карточки

Карточка может быть закрыта:

* кнопкой «×»;
* нажатием на затемненную область вне карточки.

---

# 8. Навигация по структуре

## 8.1 Выбор подразделения

При выборе подразделения в дереве:

* подразделение становится активным;
* выполняется перестроение организационной диаграммы;
* в правой части отображается выбранный узел и все его подчиненные элементы.

---

## 8.2 Разворачивание дерева

Пользователь может:

* разворачивать подразделения;
* сворачивать подразделения;
* просматривать структуру любого уровня вложенности.

Корневой узел дерева раскрыт по умолчанию.

Для подразделений с дочерними элементами отображается стрелка раскрытия.

Подразделение можно раскрыть или свернуть:

* нажатием на стрелку раскрытия;
* двойным нажатием на название подразделения.

---

## 8.3 Визуальное выравнивание дерева

Элементы дерева в левой панели должны быть визуально выровнены:

* стрелки раскрытия располагаются в единой вертикальной колонке;
* названия подразделений начинаются на одном уровне внутри каждого уровня вложенности;
* длинные названия подразделений переносятся без смещения стрелок и соседних элементов.

---

# 9. Дизайн карточек подразделений

## 9.1 Переключатель дизайна

В боковой панели доступен переключатель дизайна карточек:

```text
Дизайн карточек
```

Доступные варианты:

```text
Текущий
Вариант 2 — цветная шапка
Вариант 3 — руководитель
```

При изменении значения организационная диаграмма перестраивается с выбранным дизайном карточек.

---

## 9.2 Сохранение выбранного дизайна

Выбранный дизайн карточек сохраняется в локальном хранилище браузера:

```text
localStorage
```

Ключ хранения:

```text
orgCardDesign
```

При повторном открытии приложения используется ранее выбранный дизайн.

---

## 9.3 Текущий дизайн карточек

В текущем дизайне карточка подразделения содержит:

* название подразделения;
* ФИО руководителя;
* должность руководителя;
* численность.

При включенном режиме «Показывать вакансии» численность подсвечивается синим цветом.

---

## 9.4 Вариант 2 — цветная шапка

В варианте 2 карточка подразделения содержит:

* цветную шапку с названием подразделения;
* блок с ФИО руководителя;
* должность руководителя;
* нижний блок с численностью.

Численность отображается в формате:

```text
N сотрудников
```

---

## 9.5 Вариант 3 — руководитель

В варианте 3 карточка подразделения содержит:

* аватар с инициалами руководителя;
* название подразделения;
* ФИО руководителя;
* должность руководителя;
* численность.

Инициалы руководителя формируются автоматически из ФИО.

Если ФИО руководителя отсутствует, для аватара используется значение:

```text
—
```

Численность отображается в формате:

```text
N сотрудников
```

---

# 10. Экспорт в PDF

Приложение поддерживает экспорт текущей организационной диаграммы в PDF.

Экспорт выполняется по кнопке:

```text
📄 Экспорт в PDF
```

При экспорте:

1. формируется изображение текущей диаграммы;
2. создается PDF-документ альбомной ориентации;
3. файл сохраняется под именем:

```text
orgchart.pdf
```

Если диаграмма отсутствует, пользователю отображается сообщение:

```text
Нет диаграммы для экспорта
```

Если экспорт завершился ошибкой, пользователю отображается сообщение об ошибке.

---

# 11. Обработка загрузки и ошибок

## 11.1 Загрузка данных

При запуске приложения в левой панели отображается сообщение:

```text
Загрузка организационной структуры...
```

После успешной загрузки данных строятся:

* левое дерево подразделений;
* организационная диаграмма;
* виртуальный корневой узел.

---

## 11.2 Пустой результат загрузки

Если структура не была загружена или API вернул пустой результат, отображается сообщение:

```text
Не удалось загрузить структуру.
```

---

## 11.3 Ошибка загрузки данных

Если загрузка данных завершилась ошибкой, пользователю отображается сообщение:

```text
Не удалось загрузить организационную структуру. Проверьте доступность сервера.
```

Если ошибка произошла при инициализации приложения, в левой панели отображается сообщение:

```text
Произошла ошибка. Обновите страницу.
```

---

# 12. Адаптивность интерфейса

Приложение поддерживает адаптивное отображение для узких экранов.

При ширине экрана до 700 пикселей:

* основная компоновка меняется с горизонтальной на вертикальную;
* боковая панель занимает верхнюю часть экрана;
* панель диаграммы занимает нижнюю часть экрана;
* высота боковой панели ограничивается частью высоты экрана.

---

# 13. Итоговые возможности системы

Текущая версия приложения позволяет:

* отображать организационную структуру холдинга LEGENDA;
* использовать левое дерево только для навигации по подразделениям;
* не отображать в левом дереве руководителей, сотрудников, вакансии и численность;
* отображать руководителей подразделений вместе с должностями в организационной диаграмме;
* исключать дублирование руководителей в списках сотрудников;
* сокращать отображаемые должности до части перед символом `/`;
* отображать сотрудников и вакансии в организационной диаграмме;
* скрывать и показывать вакансии;
* переключать отображаемую численность между сотрудниками и полной численностью с учетом вакансий;
* подсвечивать численность синим цветом в текущем дизайне карточек при включенном режиме «Показывать вакансии»;
* рассчитывать численность по текущему подразделению и всем дочерним подразделениям;
* учитывать в фактической численности только сотрудников с `count >= 1`;
* сортировать сотрудников по уровню должности;
* отображать вакансии после сотрудников;
* отображать проекты сотрудников;
* открывать детальные карточки сотрудников из организационной диаграммы;
* отображать фотографии и контактные данные сотрудников;
* отображать тип занятости и статус сотрудника;
* использовать заглушки при отсутствии названия подразделения, имени сотрудника, руководителя или фотографии;
* раскрывать и сворачивать дерево подразделений;
* раскрывать и сворачивать подразделения двойным нажатием по названию;
* выбирать один из трех дизайнов карточек подразделений;
* сохранять выбранный дизайн карточек в браузере;
* экспортировать организационную диаграмму в PDF;
* отображать сообщения о загрузке и ошибках;
* адаптировать интерфейс под узкие экраны.
